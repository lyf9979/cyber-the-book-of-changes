# 赛博周易 · 八字命理分析 Skill

一个基于传统子平术（四柱八字）的结构化命理分析 skill，现已扩展为多场景命理工具箱。核心方式是：先排盘，再按用户场景加载对应参考文档，最终默认且必须生成可打开的 HTML 文件。

## 使用原则

1. **纯命理视角**：分析只依据用户提供的生辰八字，不掺杂用户其他背景信息。若用户明确要求结合其个人情况，再另行展开。
2. **文化属性声明**：首次输出时简短说明这是传统文化解读，仅供参考，不能替代理性决策。不需要反复声明，避免冗余。
3. **全维度覆盖**：每次分析必须覆盖十个维度，不可省略。
4. **可追溯**：每个结论要能追溯到命盘中的具体干支组合，避免"玄学黑话"。
5. **近三年趋势**：分析末尾必须输出基于当前流年大运的三年具体建议。
6. **HTML 文件默认输出**：无论使用哪个 reference、哪个场景模式，最终结果都必须按照 `references/html-template.md` 生成可打开的 `.html` 文件。只有用户明确要求“Markdown / 纯文本 / 调试文本”时，才允许改用 `references/output-template.md`。
7. **无环境降级**：`scripts/bazi_calculator.py` 只是高精度排盘增强工具，不是必需依赖。不得为了普通用户反复安装 Python、C++ 编译环境或 `sxtwl`；本地工具最多尝试一次，失败后立即降级。
8. **深度优先**：核心章节不得简写。十大维度每项至少 500 字；近三年按月写；总结必须带执行清单。
9. **场景分流**：根据用户意图选择单人分析、合婚、择日、生育、图像、起名、择时、顾问等模式。
10. **合规优先**：涉及婚姻、生育、健康、财务、面相、梦境时，必须使用概率性表达，并给出现实决策提醒。

## 工作流程

### 第零步：判断模式

先判断用户请求属于哪一种模式，再加载对应文档：

| 模式 | 触发词 | 必读文档 |
|---|---|---|
| `single_bazi` 单人八字 | 算命、批八字、命盘、事业、感情、财运 | `bazi-calculation.md`、`yongshen-rules.md`、`ten-dimensions.md`、`liunian-analysis.md`、`html-template.md` |
| `compatibility` 姻缘合婚 | 合婚、姻缘、配不配、适不适合结婚 | `compatibility-rules.md`、`couple-dimensions.md` |
| `wedding_date` 结婚择日 | 结婚吉日、领证日、订婚日、婚期 | `wedding-selection.md` |
| `child_planning` 生育规划 | 备孕、宝宝属相、孩子月份、出生时辰 | `child-conception.md`、`child-dimensions.md` |
| `palm_face` 手相面相 | 手相、面相、看照片、掌纹 | `palm-face-rules.md` |
| `quick_flow` 流年速测 | 今年怎么样、本月运势、快速看下 | `quick-flow.md`、`liunian-analysis.md` |
| `naming` 起名改名 | 起名、宝宝名字、改名、艺名、笔名 | `naming-rules.md`、`name-database.md` |
| `fengshui` 风水方位 | 家居风水、办公室、书桌、方位 | `fengshui-positions.md` |
| `career_pivot` 事业择时 | 跳槽、转型、换行业、离职、升职 | `career-pivot.md` |
| `wealth_timing` 财运择时 | 投资时机、大额支出、买房、破财 | `wealth-timing.md` |
| `taisui` 太岁本命年 | 本命年、犯太岁、冲太岁 | `taisui-rules.md` |
| `dream` 梦境综合 | 做梦、梦见、反复梦境 | `dream-symbols.md` |
| `consultation` 顾问问答 | 继续问、帮我判断 A/B、下个月适合吗 | 复用已生成报告和相关文档 |

若一个请求同时命中多个模式，优先处理用户最明确的目标。例如“我们合婚并看结婚日”先做 `compatibility`，再做 `wedding_date`。

### 第一步：收集与核验信息

需要用户提供：
- 出生日期（公历/农历需注明，默认公历）
- 出生时间（精确到时；若用户只给"上午""下午"等模糊说法，要追问时辰）
- 性别（决定大运顺逆排）
- 出生地（可选，用于真太阳时校正；若未提供则使用标准时）

若信息不全，**先追问再开始**。子时分早晚子时要特别确认（00:00–01:00 为早子时归次日，部分流派归当日）。本 skill 默认采用 **00:00 之后归次日子时** 的现代通行算法。

不同模式额外信息：
- 合婚：需要双方完整生辰；可选关系阶段。
- 择日：需要双方八字和目标日期范围。
- 生育规划：需要父母双方八字和目标年份；不能处理性别选择。
- 手相/面相：需要八字和清晰图片；未成年人照片不分析。
- 起名：需要姓氏、八字、风格偏好和避讳字。
- 风水：需要房屋朝向或空间描述。
- 择时：需要计划类型与目标时间范围。

### 第二步：排盘

按以下顺序计算并列出：

1. **四柱**：年柱 / 月柱 / 日柱 / 时柱（天干地支）
2. **藏干**：每个地支的本气、中气、余气
3. **十神**：以日干为中心，标注每个天干与藏干的十神关系
4. **五行统计**：金木水火土各自的分数/出现次数
5. **大运**：起运岁数 + 未来 7–8 步大运
6. **神煞**（可选，仅标注影响大的几个）：桃花、华盖、驿马、羊刃、天乙贵人等

排盘过程必须展示给用户，便于核对。

> 详细排盘算法与规则见 `references/bazi-calculation.md`

排盘来源必须在 HTML 的“基本信息卡”或“排盘口径”中标明：

- `sxtwl local`：本地可用 `scripts/bazi_calculator.py` 且一次调用成功，属于高精度排盘。
- `user provided`：用户已经给出完整四柱，按用户四柱分析，并提示“未重新校验历法”。
- `LLM approximate`：本地工具不可用且用户未提供四柱时，按 reference 规则进行近似排盘，必须标注“近似排盘，仅供文化参考”。

如果本地脚本缺少 Python、C++ 构建环境或 `sxtwl`，不得循环安装、不得反复试错。停止工具调用，进入降级排盘，并在报告中写明来源与置信度。

### 第三步：格局与用神判定

1. 判断日主强弱（得令 / 得地 / 得生 / 得助）
2. 确定格局类型（正格：正官、七杀、正财、偏财、正印、偏印、食神、伤官；或特殊格局）
3. 确定喜用神与忌神
4. 列出命中"通关字""调候字""救应字"

> 详细判定规则见 `references/yongshen-rules.md`

### 第四步：十大维度全面分析

每个维度都要基于命盘的具体干支结构作答，严禁套话。顺序固定如下：

1. **性格特质** —— 依据日主 + 日支 + 月令 + 透干十神
2. **感情运势** —— 男看财星 / 女看官杀，结合配偶宫（日支）
3. **家庭与六亲** —— 年柱看祖辈、月柱看父母兄弟、日柱看自身与配偶、时柱看子女
4. **财富格局** —— 财星旺衰 + 身财对比 + 财星位置 + 食伤生财
5. **健康倾向** —— 五行偏枯对应脏腑 + 冲克刑害的影响
6. **事业方向** —— 喜用神五行对应行业 + 官杀印食伤的组合 + 格局类型
7. **配偶特征** —— 配偶宫（日支）+ 配偶星（男财女官）的字性、位置、喜忌
8. **居住与风水** —— 喜用神方位、颜色、环境要素（火喜南向红色暖色、水喜北向黑蓝等）
9. **学业 / 考试 / 名气** —— 印星、文昌、官星组合 + 食神吐秀
10. **人生事件节点** —— 基于大运流年的重大节点预测（婚、迁、变、成）

> 每个维度的详细分析框架见 `references/ten-dimensions.md`
> 输出深度要求见 `references/output-template.md`（仅作内容结构参考；默认最终呈现仍使用 HTML，每个维度至少 500 字）

### 第五步：近三年发展趋势建议（2026–2028）

必须输出 2026 / 2027 / 2028 三年趋势，且每年按 12 个月逐月总结。

对每一年分析：
- 天干地支与日主/用神的关系
- 对十大维度中影响最大的 3–4 个维度的具体预测
- **逐月建议表（1–12 月）**：每月包含主线、风险、行动、优先级
- **可操作的行动建议**（宜做什么、避免什么、时间窗口）

> 流年分析方法见 `references/liunian-analysis.md`

### 第六步：输出格式

- 默认：严格按照 `references/html-template.md` 生成完整单文件 HTML，从 `<!doctype html>` 开始。
- 文件落盘：在具备文件系统能力的 Agent 中，必须将报告保存为 `.html` 文件，优先使用 `outputs/<YYYYMMDD-HHMMSS>-<mode>-report.html`。
- 控制台路径提示：文件生成后必须在控制台/最终回复中输出 `HTML 报告已生成：<文件路径>`，优先使用绝对路径，方便用户直接复制打开。不要把整段 HTML 粘贴到聊天/控制台里。
- 全场景一致：无论加载 `compatibility`、`wedding_date`、`naming`、`fengshui`、`dream` 等任一 reference，最终都必须包成可打开的 HTML 文件，不在控制台式 Markdown 中结束。
- 无文件系统兜底：若平台不能写入文件，则创建平台支持的 HTML Artifact / Canvas / 附件；若仍不支持，才输出一个带文件名提示的完整 HTML 代码块，并明确建议保存为 `.html`。
- 文本例外：只有用户明确要求“Markdown / 纯文本 / 调试文本 / 不要 HTML”时，才改用 `references/output-template.md`。
- 完整性：HTML 必须保证章节完整、结论可回溯到命盘，不能因为换成网页而删减内容。
- **总结章节必须包含“总结与执行清单”**（30 天 / 90 天 / 年度执行计划 + 复盘机制）

## 扩展模式输出要求

### 姻缘合婚

输出双方命盘摘要、总分、8 维关系分析、协同点、摩擦点、未来三年共同节点、关系经营建议。禁止替用户决定结婚或分手。

### 结婚择日

输出日期范围、Top 10 推荐日期、月度热度表、避开日期、现实档期提醒。若缺少目标范围，先追问。

### 生育规划

输出推荐出生月份、受孕窗口区间、属相合冲、候选月份对比、子女简化十维、父母经营建议。严禁性别选择和医学判断。

### 手相 / 面相

先描述客观可见特征，再给出与八字的“印证 / 补充 / 矛盾”标签。图片不清晰时只做有限分析并提示重拍。

### 起名 / 改名

输出 5–10 个候选名，每个名字包含字义、五行补益、音律、与姓氏组合、风险提示。

### 事业 / 财运择时

输出未来 12–24 个月窗口表，标注绿/黄/红，并给出准备清单。必须声明不构成职业或投资建议。

### 风水方位

输出喜用方位、家居/办公/卧室建议、颜色材质建议、避免模式。不推荐购买任何风水物品。

### 太岁 / 本命年

输出未来 12 年太岁关系时间线和现实行为清单。不提供护符、开光、化煞商品建议。

### 梦境综合

输出梦境符号、当前流年流月关联、现实提醒。若用户表达自伤或严重心理困扰，引导寻求专业帮助。

## 关键禁忌

- **不做绝对预言**：使用"倾向于""传统上看""命理角度认为"等表述
- **不涉及生死大病**：健康部分只提倾向，不下诊断
- **不替代专业决策**：涉及法律、医疗、重大投资时提醒用户寻求专业人士
- **不判定迷信行为**：不推荐风水改运、开光、化煞等具体消费行为
- **对敏感话题保持克制**：涉及父母寿元、婚变、破财等负面推断时，点到为止，侧重"如何规避"而非"何时发生"
- **不做性别选择**：生育规划不预测、不引导、不暗示胎儿性别。
- **不评价容貌优劣**：面相只做传统文化倾向分析，不做外貌评判。

详见项目根目录 `AGENTS.md` 与 `docs/integration.md`。
